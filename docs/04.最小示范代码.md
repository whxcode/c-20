# 04.最小示范代码

这份文档给出一个可学习的最小 C++ 骨架，保留核心思想：统一 setter、collector 暂存、commit 生成 undo/redo、history 或远端 patch 回放时按场景控制收集。

## 最小数据结构

```cpp
enum class PropKey { Name, X, Y };
enum class PatchType { Update, Create, Delete };

struct PatchItem {
    PatchType type = PatchType::Update;
    int id = 0;
    std::unordered_map<PropKey, std::any> props;
};

struct PatchPair {
    std::vector<PatchItem> undo;
    std::vector<PatchItem> redo;
};
```

备注：

- `PatchItem` 是纯数据，不持有 model 指针，方便序列化和网络传输。
- 示例只覆盖 `Update`，真实项目还需要处理 `Create/Delete/Move`。
- `std::any` 只是示范，生产代码通常会使用可序列化的 variant 或 schema 类型。

## Collector

```cpp
class Collector {
public:
    void close() { ++disabledDepth_; }

    void open() {
        if (disabledDepth_ > 0) {
            --disabledDepth_;
        }
    }

    bool canCollect() const { return enabled_ && disabledDepth_ == 0; }

    template <class T>
    void dataChanged(int id, PropKey key, const T& oldValue, const T& newValue) {
        if (!canCollect()) return;

        auto& item = items_[id];
        item.id = id;

        if (!item.used.contains(key)) {
            item.oldProps[key] = oldValue;
            item.used.insert(key);
        }
        item.newProps[key] = newValue;
    }

    std::optional<PatchPair> commit() {
        if (items_.empty()) return std::nullopt;

        PatchPair pair;
        for (auto& [id, item] : items_) {
            pair.undo.push_back({PatchType::Update, id, item.oldProps});
            pair.redo.push_back({PatchType::Update, id, item.newProps});
        }
        items_.clear();
        return pair;
    }

private:
    struct Item {
        int id = 0;
        std::unordered_set<PropKey> used;
        std::unordered_map<PropKey, std::any> oldProps;
        std::unordered_map<PropKey, std::any> newProps;
    };

    bool enabled_ = true;
    int disabledDepth_ = 0;
    std::unordered_map<int, Item> items_;
};
```

备注：

- 同一轮 commit 内，同一属性多次修改时，`oldProps` 只保留第一次修改前的值。
- `newProps` 始终更新为最后一次修改后的值。
- `close/open` 使用深度计数，避免嵌套关闭时提前恢复收集。

## Model

```cpp
class Model {
public:
    explicit Model(int id, Collector* collector) : id_(id), collector_(collector) {}

    void setName(std::string value) {
        if (name_ == value) return;
        auto oldValue = name_;
        name_ = std::move(value);
        triggerChanged(PropKey::Name, oldValue, name_);
    }

    template <class T>
    void triggerChanged(PropKey key, const T& oldValue, const T& newValue) {
        if (listener_) listener_(key);
        if (collector_) collector_->dataChanged(id_, key, oldValue, newValue);
    }

private:
    int id_ = 0;
    std::string name_;
    Collector* collector_ = nullptr;
    std::function<void(PropKey)> listener_;
};
```

备注：

- 所有可 undo 的修改必须走 setter。
- setter 必须先保存 oldValue，再写入 newValue，最后触发 `triggerChanged`。
- listener 负责通知 UI/组件树；collector 负责记录历史，不要混成同一个职责。

## CommitService

```cpp
class CommitService {
public:
    void commit() {
        document_.triggerUpdated();
        auto pair = collector_.commit();
        if (pair) {
            history_.push(*pair);
            sendToRemote(pair->redo);
        }
    }

    void applyRemotePatch(const std::vector<PatchItem>& redo) {
        collector_.close();
        mergePatch(redo);
        collector_.open();
        document_.triggerUpdated();
    }

    void undo() {
        auto patch = history_.takeUndo();
        if (!patch) return;

        collector_.close();
        mergePatch(patch->undo);
        collector_.open();
        document_.triggerUpdated();
    }
};
```

备注：

- `commit()` 前先执行 `document_.triggerUpdated()`，确保布局、副作用、派生修改也进入同一轮提交。
- `applyRemotePatch()` 关闭 collector，避免远端 patch 被再次记录成本地操作并回发。
- 示例里 `undo()` 也关闭了 collector，这是最容易理解的做法；真实项目如果需要收集 undo/redo 产生的文档副作用，可以按执行模式决定是否关闭 collector。

## 实现原则

1. 所有可 undo 的状态修改必须走统一 setter。
2. setter 必须在写入前保存 oldValue，写入后拿到 newValue。
3. collector 以 “model id + prop key” 为最小记录单元。
4. 同一 commit 内同属性多次修改，只保留第一次 old 和最后一次 new。
5. commit 前要先跑文档副作用，否则副作用不会进历史。
6. undo/redo/远端 patch 合并时要区分是否允许收集。
7. 远端 IO patch 通常关闭 collector，避免回声循环。
8. HistoryManager 不应直接依赖 Document；用 executor 回调解耦。
9. Patch 应尽量是可序列化的纯数据，而不是函数或对象指针。
