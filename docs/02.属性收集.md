## 属性修改收集

修改属性时：

```text
setXxx()
  -> oldValue = 当前值
  -> 写入新值
  -> triggerChanged(scope, key, &oldValue, &newValue)
  -> fCollector->dataChanged(id, type, key, oldValue, newValue)
```

关键代码：

- `mk-provider/src/models/MkModelBase.cpp:82`：先通知绑定组件，再进入属性收集。
- `mk-provider/src/models/MkModelBase.cpp:88`：collector 可用时调用 `dataChanged()`。
- `mk-provider/src/collector/MkCollectorImpl.cpp:16`：`dataChanged()` 入口。
- `mk-provider/src/collector/MkCollectorImpl.cpp:32`：按 model id 找或创建 `MkCollectorItem`。
- `mk-provider/src/collector/MkCollectorImpl.cpp:52`：把本属性 old/new 写入 item。

`MkCollectorItem::update()` 的关键策略：

- 第一次改某属性：记录旧值到 `fOldProps`，记录新值到 `fNewProps`，`fUsed[key]=true`。
- 同一轮 commit 前重复改同一属性：只更新 `fNewProps`，旧值保持第一次修改前的值。

这就是自动 undo/redo 的核心：一次 commit 内，undo 回到“第一次修改前”，redo 到“最后一次修改后”。

## Commit 生成 undo/redo

`MkCommitService::doCommit()` 是提交核心：

```cpp
std::optional<MkPatchPair> MkCommitService::doCommit(MkCommitOptions&& options) {
    const auto& document = getDocument();

    document->triggerUpdated(MkDataChangeType::kNormal);

    auto patchPair = document->getDataProvider()->getCollector()->commit();
    if (patchPair.has_value()) {
        fHistoryManager->commit(patchPair.value(), std::move(options));
    }

    return patchPair;
}
```

关键点：

- 先 `triggerUpdated()`，再 `collector->commit()`。
- 原因是触发文档更新时可能产生图层副作用；这些副作用也要被 collector 收进去。
- `collector->commit()` 会生成 `MkPatchPair{undo, redo}`，并清空暂存。

`MkCollectorImpl::commit()`：

- `mk-provider/src/collector/MkCollectorImpl.cpp:150`：无数据返回空。
- `mk-provider/src/collector/MkCollectorImpl.cpp:154`：调用 `doCommit()` 生成 patch pair。
- `mk-provider/src/collector/MkCollectorImpl.cpp:155`：清空 pool。
- `mk-provider/src/collector/MkCollectorImpl.cpp:156`：清空 `fItems`。

`MkCollectorImpl::doCommit()`：

- `mk-provider/src/collector/MkCollectorImpl.cpp:160`：创建 redo/undo 两个 `mkkiwi::Patch`。
- `mk-provider/src/collector/MkCollectorImpl.cpp:168`：遍历所有 `MkCollectorItem`。
- `mk-provider/src/collector/MkCollectorImpl.cpp:172`：每个 item 写入 undo/redo patch item。
- `mk-provider/src/collector/MkCollectorImpl.cpp:180`：编码为 `SkData` buffer。

`commitItem()` 会把 patch type 反转：

```text
正向 redo:
  Update -> newProps
  Create -> create model
  Delete -> delete model

反向 undo:
  Update -> oldProps
  Create -> delete model
  Delete -> create old model
```

## Undo/Redo 执行

调用 `MkCommitService::undo()`：

```text
MkCommitService::undo()
  -> MkHistoryManager::undo()
  -> getPatchBuffers(kUndo, entry)
  -> fOnExecutor(kUndo, undoBuffers, undoOptions)
  -> MkCommitService::onPatchExecutor()
  -> mergePatches(kUndo, buffers, false)
```

关键点：

- `MkHistoryManager` 只决定取哪条历史、取 undo 还是 redo。
- 真正应用 patch 的是 `MkCommitService::onPatchExecutor()`。
- `onPatchExecutor()` 会先撤销未提交的 staging 数据，避免当前脏数据污染历史回放。
- undo/redo 回放时 `mergePatches(mode, patches, false)` 传 `false`，表示不关闭 collector，要让回放产生的文档副作用也能被收集。

代码位置：

- `mk-editor/src/service/MkCommitService.cpp:231`：`onPatchExecutor()`。
- `mk-editor/src/service/MkCommitService.cpp:240`：先提交并回滚 staging patch。
- `mk-editor/src/service/MkCommitService.cpp:254`：应用历史 patch。
- `mk-editor/src/service/MkCommitService.cpp:294`：undo 入口。
- `mk-editor/src/service/MkCommitService.cpp:299`：redo 入口。

## Patch 写回 Model 和组件树

Patch 应用由 `MkMergePatch` 完成：

- `mk-editor/src/service/MkCommitService.cpp:336`：`mergePatches()` 创建 `MkDocumentMergeDelegate`。
- `mk-provider/src/state/MkMergePatch.cpp:33`：`merge()` 可选择关闭 collector。
- `mk-provider/src/state/MkMergePatch.cpp:61`：解码 patch items。
- `mk-provider/src/state/MkMergePatch.cpp:121`：update patch 入口。
- `mk-provider/src/state/MkMergePatch.cpp:136`：找到 model 并按 `used` 字段逐个属性写回。

update 写回 Model 时，走的是更底层的 `propAttr->props()->setValue(model.get(), value)`，而不是外层 `MkModelBase::setValue()`。这仍会进入具体属性 setter，因此会触发：

```text
Model setter
  -> triggerChanged
  -> component listener
  -> MkComponent::onDataChanged
  -> OperationCollector::modifyOperation
```

组件观察者链路在：

- `mk-gui/src/graphics/MkComponent.cpp:62`：组件构造/换 model 时注册 `setChangedListener`。
- `mk-gui/src/graphics/MkComponent.cpp:184`：model changed callback 转为 `onDataChanged()`。
- `mk-gui/src/graphics/MkComponent.cpp:216`：触发组件数据版本变化。
- `mk-gui/src/graphics/MkComponent.cpp:220`：通知 observers 数据改变。
